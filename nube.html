5. Desarrollo de Aplicaciones en la Nube

5.1 Concepto de Computación en la Nube

La computación en la nube es un modelo que permite el acceso bajo demanda a recursos tecnológicos a través de Internet.

Bajo demanda

Recursos disponibles en cualquier momento vía Internet.

Acceso global

Disponible desde cualquier lugar del mundo.

Pago por uso

Solo se paga por los recursos que se consumen.

5.2 Modelos de Servicio

Modelo Descripción Ejemplo
IaaS Infraestructura como servicio. Proporciona servidores, redes y almacenamiento virtualizados. AWS EC2
PaaS Plataforma como servicio. Entorno listo para desarrollar y desplegar aplicaciones. Azure App Service
SaaS Software como servicio. Aplicaciones completas accesibles desde el navegador. Google Workspace
IaaS

El proveedor gestiona la infraestructura física. El usuario controla SO, middleware y aplicaciones.

PaaS

El proveedor gestiona infraestructura y plataforma. El usuario solo desarrolla y despliega su código.

SaaS

El proveedor gestiona todo. El usuario accede al software directamente sin instalaciones.

5.3 Escalabilidad

La nube permite escalabilidad horizontal y vertical según la demanda del sistema.

Escalabilidad Vertical

Aumenta los recursos de un mismo servidor (más CPU, RAM).

Escalabilidad Horizontal

Añade más instancias del servidor para distribuir la carga.

5.4 Alta Disponibilidad

Los proveedores de nube garantizan disponibilidad mediante replicación y balanceo de carga.

Replicación

Los datos se copian en múltiples ubicaciones geográficas.

Balanceo de carga

Distribuye el tráfico entre múltiples servidores.

Failover automático

Si un nodo falla, el tráfico se redirige automáticamente.

5.5 Ventajas del Desarrollo en la Nube

Reducción de costos iniciales

Sin inversión en infraestructura física propia.

Escalabilidad automática

Los recursos se ajustan según la demanda en tiempo real.

Acceso global

Disponible desde cualquier dispositivo con conexión a Internet.

Seguridad avanzada

Cifrado, autenticación y cumplimiento normativo integrados.

5.6 Relación con APIs y Microservicios

Las APIs y microservicios suelen desplegarse en la nube para garantizar disponibilidad y escalabilidad global.

Web APIs

Se exponen como endpoints globales accesibles desde cualquier cliente.

Microservicios

Cada servicio escala independientemente en la nube según su carga.

Contenedores

Se orquestan en la nube con herramientas como Kubernetes.
