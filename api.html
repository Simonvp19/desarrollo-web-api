1. Tecnologías de Web APIs

1.1 Definición de API

Una API (Application Programming Interface) es un conjunto de reglas y protocolos que permiten la comunicación entre diferentes sistemas de software. Funciona como intermediaria para facilitar el intercambio estructurado de datos.

Las Web APIs utilizan el protocolo HTTP para enviar y recibir solicitudes entre cliente y servidor, generalmente en formato JSON.

Comunicación

Facilita el intercambio de datos entre sistemas distintos.

HTTP

Usa el protocolo HTTP como base de comunicación.

JSON

Los datos se intercambian en formato JSON.

1.2 Relación entre API y HTTP

Las Web APIs generalmente utilizan el protocolo HTTP (HyperText Transfer Protocol) para intercambiar información entre cliente y servidor.

HTTP define métodos que permiten realizar operaciones sobre recursos:

Método Función
GETObtener información
POSTEnviar o crear información
PUTActualizar datos existentes
DELETEEliminar información

Gracias a HTTP, las APIs pueden funcionar de manera estandarizada a través de internet.

1.3 Funcionamiento de una Web API

Las Web APIs operan mediante solicitudes HTTP que permiten realizar acciones sobre recursos específicos en un servidor.

Método HTTP Descripción Técnica Uso Común Ejemplo
GET Solicita información sin modificar datos. Consultar registros /api/usuarios
POST Envía datos para crear un recurso nuevo. Registrar usuario /api/usuarios
PUT Actualiza información existente. Modificar usuario /api/usuarios/1
DELETE Elimina un recurso del sistema. Borrar registro /api/usuarios/1

Códigos de Estado HTTP

HTTP utiliza códigos de estado para indicar el resultado de una solicitud. Permiten saber si la operación fue exitosa o si ocurrió algún error.

Código Significado Descripción
200 OKÉxitoLa solicitud fue procesada correctamente.
201 CreatedCreadoEl recurso fue creado exitosamente.
400 Bad RequestError del clienteLa solicitud contiene datos incorrectos.
401 UnauthorizedNo autorizadoSe requiere autenticación.
404 Not FoundNo encontradoEl recurso solicitado no existe.
500 Server ErrorError del servidorOcurrió un problema en el servidor.

Los códigos de estado son fundamentales para manejar respuestas del servidor correctamente.

1xx Informativas
2xx Exitosas
3xx Redirecciones
4xx Error del cliente
5xx Error del servidor

1.4 Arquitectura REST

La arquitectura REST (Representational State Transfer) es el modelo más utilizado en el desarrollo de Web APIs modernas.

Protocolo estándar HTTP

Comunicación basada en métodos HTTP ampliamente adoptados.

Sin estado (Stateless)

Cada solicitud contiene toda la información necesaria.

Recursos por URL

Cada recurso se identifica mediante una URL única.

Datos en JSON

Representación de recursos en formato JSON estándar.

1.6 Seguridad en Web APIs

La seguridad en las APIs es fundamental para proteger los datos intercambiados. Entre los mecanismos más utilizados se encuentran:

JWT (JSON Web Token)

Autenticación segura mediante tokens firmados digitalmente.

OAuth 2.0

Protocolo de autorización estándar en la industria.

HTTPS

Cifrado de datos en tránsito mediante TLS/SSL.

Control de roles

Acceso basado en roles y permisos de usuario.

Ventajas de las Web APIs

Interoperabilidad

Conexión entre sistemas heterogéneos.

Escalabilidad

Crece según las necesidades del sistema.

Reutilización

Servicios compartidos entre proyectos.

Mantenimiento

Más fácil de mantener y actualizar.

Integración

Compatible con apps móviles y web.
