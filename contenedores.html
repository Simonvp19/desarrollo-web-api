4. Contenedores y Virtualización

4.1 ¿Qué es un contenedor?

Un contenedor es una unidad ligera y portable que incluye una aplicación y todas sus dependencias necesarias para ejecutarse de manera consistente en cualquier entorno.

Ligero

Comparte el kernel del sistema operativo anfitrión.

Portable

Se ejecuta de forma idéntica en cualquier entorno.

Aislado

Cada contenedor opera de forma independiente.

4.2 Diferencia entre Máquina Virtual y Contenedor

Aspecto Máquina Virtual Contenedor
Sistema Operativo Incluye SO completo Comparte el SO del host
Uso de recursos Mayor consumo Ligero y eficiente
Tiempo de arranque Lento Rápido

4.3 Docker como herramienta principal

Docker es la plataforma más utilizada para la creación y gestión de contenedores. Permite empaquetar aplicaciones en imágenes que pueden ejecutarse en cualquier entorno.

Imágenes Docker

Plantillas inmutables con todo lo necesario para ejecutar una aplicación.

Contenedores en ejecución

Instancias activas de una imagen Docker corriendo en el host.

4.4 Componentes de Docker

Docker Client

Interfaz de línea de comandos para interactuar con Docker.

Docker Daemon

Proceso en segundo plano que gestiona contenedores e imágenes.

Docker Registry

Repositorio para almacenar y distribuir imágenes Docker.

Imágenes

Plantillas de solo lectura para crear contenedores.

Contenedores

Instancias ejecutables e independientes de una imagen.

4.5 Ventajas del uso de contenedores

Portabilidad entre entornos

Funciona igual en desarrollo, pruebas y producción.

Consistencia en despliegues

Elimina el problema de "funciona en mi máquina".

Escalabilidad rápida

Nuevos contenedores se levantan en segundos.

Optimización de recursos

Menor consumo de CPU y memoria respecto a VMs.

4.6 Relación con Microservicios

Los microservicios suelen desplegarse dentro de contenedores, lo que permite que cada servicio funcione de manera independiente y escalable.

Un servicio, un contenedor

Cada microservicio vive en su propio contenedor aislado.

Independencia y resiliencia

El fallo de un contenedor no afecta a los demás servicios.
